# 📋 Telegram Task Manager Bot

Асинхронный Telegram-бот для управления задачами с продвинутой архитектурой.  
Создан в учебных целях и демонстрации навыков работы с `aiogram 3.x`, `SQLAlchemy 2.0` и асинхронным Python.

---

## 🚀 Возможности

### 📌 Управление задачами
- **Добавление задачи** (`/add`) с пошаговым мастером (FSM):
  - *Название* — обязательное поле, любое текстовое описание.
  - *Описание* — подробности, можно пропустить (ключевое слово `пропустить`).
  - *Дедлайн* — дата и время в формате `ГГГГ-ММ-ДД ЧЧ:ММ`, с проверкой корректности; можно пропустить.
- **Просмотр списка** (`/list`) с умной пагинацией:
  - Задачи выводятся порциями по 5 штук.
  - Каждая задача отмечена иконкой статуса (✅ выполнена / ⬜ в процессе).
  - Кнопки «◀️ Назад» / «Вперёд ▶️» для листания страниц.
- **Детальный просмотр** — при клике на задачу открывается карточка с полной информацией: название, описание, дедлайн, статус.
- **Быстрые действия** прямо из карточки задачи:
  - ✔️ *Выполнено* — отметить задачу завершённой (мгновенно обновляет статус).
  - ❌ *Удалить* — безвозвратно удалить задачу с подтверждением.
- **Статистика** (`/stats`):
  - Общее количество задач.
  - Количество выполненных.
  - Количество оставшихся.

### 🛡 Обработка ошибок и удобство пользователя
- **Валидация ввода** на каждом шаге создания задачи.
  - При неверном формате дедлайна бот вежливо просит повторить ввод или написать `пропустить`.
- **Защита от чужих задач** — каждый пользователь видит только свои данные.
- **Информативные сообщения** при отсутствии задач или попытке выполнить действие над несуществующей задачей.

### ⚙️ Архитектурные особенности (для разработчиков)
- **FSM (машина состояний)** – диалог добавления задачи реализован через `aiogram.fsm`, состояния хранятся в памяти (легко перенести в Redis).
- **Middleware для базы данных** – асинхронная сессия SQLAlchemy автоматически пробрасывается в каждый обработчик, не нужно вручную открывать/закрывать соединение.
- **Полностью асинхронный код** – бот не блокируется при работе с базой или ожидании ответа от Telegram.
- **Разделение кода** – обработчики, клавиатуры, модели БД и middleware вынесены в отдельные модули, что упрощает поддержку и расширение.
- **Готовность к вебхукам** – переход с `polling` на `webhook` потребует минимальных изменений.

## 🛠 Технологии

| Компонент | Инструмент |
|-----------|------------|
| Язык | Python 3.12 |
| Фреймворк | [Aiogram 3.x](https://docs.aiogram.dev/) |
| База данных | SQLite + [SQLAlchemy 2.0](https://docs.sqlalchemy.org/) (async) |
| Хранение состояний | Встроенный FSM (память) |
| Логирование | Стандартный `logging` |
| Деплой | Возможен на любой VPS, поддержка вебхуков (в перспективе) |

## 📁 Структура проекта

```
todo-bot/
├── bot.py # точка входа, инициализация диспетчера
├── config.py # токен бота (не загружается в репозиторий)
├── config.example.py # пример конфигурации для быстрого старта
├── requirements.txt # зависимости
├── .gitignore # скрываем config.py и временные файлы
├── database/
│ ├── engine.py # асинхронный движок и создание таблиц
│ └── models.py # модель Task (SQLAlchemy)
├── handlers/
│ ├── start.py # /start
│ ├── add_task.py # /add и машина состояний
│ ├── list_tasks.py # /list с пагинацией и колбэками
│ ├── done.py # отметка выполнения
│ ├── delete.py # удаление задачи
│ └── stats.py # /stats
├── keyboards/
│ └── inline.py # инлайн‑клавиатуры для списка и действий
├── middlewares/
│ └── db_session.py # middleware для внедрения сессии БД
└── states/
└── task_states.py # состояния для FSM добавления задачи
```

## ⚙️ Установка и запуск

### 1. Клонирование репозитория
```bash
git clone https://github.com/твой_логин/todo-bot.git
cd todo-bot

python -m venv venv
source venv/bin/activate      # Linux / macOS
venv\Scripts\activate         # Windows

pip install -r requirements.txt

cp config.example.py config.py   # Linux / macOS
copy config.example.py config.py # Windows

BOT_TOKEN = "ваш_токен_от_BotFather"

python bot.py
